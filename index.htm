Welcome to my online portfolio. This page primarily features a bulletin of changes and pending inclusions to my "Projects" section of the website. Enjoy.

September 5, 2020: Article Added: Moving to Parallelism with Embarrassingly Parallel Problems

An article has been added in Projects/Parallelism/Moving a Serial Problem to Parallel: Embarrassingly Parallel Problems. Recently I have been preparing to build a new physics engine. Along the way I was building new serial and parallel methods to chain/island sets of interacting objects in my API for simulations. Describing this task offers a useful time to discuss an easy way to transition to parallel thinking.

August 10, 2020: Article Added: Non-Iterative Position Projection

An article has been added in Projects/One Pass Mechanics/Using Time of Solving and Speculative Contacts with Position Projection. I have worked for a few years on avoiding contact resolution with an iterative constraint solver, but this more recent iteration involves replacing some processing overhead with speculative contacts.

July 13, 2020: Article Added: Performant Java Code

An article has been added in Projects/Specialized Optimizations/Asynchronous Behavior in Java. It discusses when OOP abstractions might not offer a programmer an ideal tool to implement in a specific project.

June 12, 2020: Parallel Projects Area Added

An area for projects relating to parallelism and concurrency has been added. The first entry is a series of tests that illustrate performance differences in software when applied to a CPU and a GPU.

May 25, 2020: Analysis of Code Branching on CPU vs. GPU

This Friday, information pertaining to an analysis of tests run between parallel workloads on a GPU and a CPU will be provided in the Projects area. This will be in a new category for "Parallel Workflow" and will have further projects included to describe problems faced with race conditions and mutual exclusion among other topics.

May 24, 2020: Fixing Some Things

The site has been rearranged to offer categories within the projects area. This is to expand work that has been performed in a common subject - or what is at least related.
